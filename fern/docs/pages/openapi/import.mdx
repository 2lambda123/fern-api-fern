To use an OpenAPI specification, you can pass in the filepath or URL.

Start by ensuring Fern is installed.

```bash
npm install -g fern-api
```

## From filepath 

<CodeBlocks>
  <CodeBlock title="Import from filepath">
  ```bash
  fern init --openapi <filepath to openapi>
  ```
  </CodeBlock>
  <CodeBlock title="Example">
    ```bash
    fern init --openapi .../api-spec/openapi.yaml # can be YAML or JSON
    ```
<CodeBlocks>

## From URL 

<CodeBlocks>
  <CodeBlock title="Import from URL">
  ```bash
  fern init --openapi <url to openapi>
  ```
  </CodeBlock>
  <CodeBlock title="Example">
    ```bash
    fern init --openapi https://raw.githubusercontent.com/OAI/OpenAPI-Specification/main/examples/v3.0/petstore.yaml # can be YAML or JSON
    ```
<CodeBlocks>

## Behavior

Fern will generate the following directory with your OpenAPI spec inside of it.

```bash
fern/
├─ fern.config.json
├─ generators.yml
└─ openapi/
  └─ openapi.json # <--- your openapi spec
```

Run `fern check` to validate the OpenAPI spec. Resolve any errors.

Then run [fern generate](/overview/cli/fern-generate).

<Callout intent="info">
If you're having trouble understanding the errors, run the command `fern write-definition`. This command will convert your OpenAPI spec into a Fern Definition. [Read more.](/overview/cli/fern-write-definition)
</Callout>
